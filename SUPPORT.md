# Support

Use the right channel so maintainers can act on the report quickly.

## Open A GitHub Issue When

- something in a public repo is broken
- docs and runtime behavior do not match
- a public example no longer works
- you found a reproducible bug or regression

## Use The Security Process When

- the report could expose a vulnerability
- the issue affects keys, signatures, auth, secrets, funds, or deployment safety

Follow [SECURITY.md](SECURITY.md) for private reporting.
